Приложение «Умный список дел» (Smart Todo)

Функциональные требования
Добавление задачи – управляемая форма, при отправке новая задача добавляется в список. Обязательные поля: заголовок (не пустой), остальные могут быть пустыми. Приоритет по умолчанию «средний».

Отметка о выполнении – чекбокс меняет done задачи, порядок сортировки не меняется.

Удаление задачи – удаляет конкретную задачу.

Редактирование задачи – по кнопке «Редактировать» открывается форма (можно внутри того же компонента или модальное окно) с предзаполненными полями, после сохранения изменения применяются.

Фильтрация – показывает задачи в зависимости от выбранного статуса (all, active, completed).

Поиск – текстовое поле, фильтрует задачи по вхождению подстроки в title (регистронезависимо).

Сохранение в localStorage – при любом изменении списка задач он сохраняется. При загрузке страницы данные читаются из localStorage; если их нет – используется начальный массив из 3–4 демо-задач.

Счётчик – показывает количество задач, у которых done === false.

Очистка выполненных – удаляет все задачи с done === true.

Дополнительно для хуков:

useEffect – для синхронизации с localStorage (при изменении tasks).

useRef – для автоматического фокуса на поле ввода при монтировании формы добавления.

useLayoutEffect – например, для измерения высоты списка задач после добавления новой (можно вывести в консоль или показать где-то размер). Или для плавной прокрутки к новой задаче.

children – сделать отдельный компонент Card или Wrapper, который принимает children и рендерит обводку/фон.

Чек-лист проверки знаний
Тема	Где применяется в проекте
JSX	Вся разметка компонентов
Props	Передача данных и колбэков от App в дочерние компоненты (TaskList, TaskItem, Filter, Search, AddTask)
Children	Компонент-обёртка (Card, Modal или Section)
map + key	Отображение списка задач (ключ – task.id)
События	onClick, onChange, onSubmit (добавление, удаление, редактирование, фильтрация, поиск)
useState	tasks, filter, searchQuery, editingTask (текущая редактируемая задача), isEditModalOpen
useEffect	Чтение из localStorage при монтировании, запись в localStorage при изменении tasks
useLayoutEffect	Пример: после добавления новой задачи измерить высоту списка или прокрутить к последней задаче (выполняется до отрисовки)
useRef	Ссылка на поле ввода заголовка в форме добавления, чтобы установить фокус при монтировании
Подъём состояния	Состояние tasks и функции-обработчики живут в App и передаются вниз
Управляемые формы	Форма добавления и форма редактирования (значения полей хранятся в state)
Неуправляемые формы (опционально)	Можно сделать форму поиска через ref, но лучше управляемую для единообразия
Пошаговый план реализации
Шаг 1. Настройка окружения
Создать проект: npm create vite@latest smart-todo -- --template react

Удалить лишние файлы, оставить App.jsx, main.jsx, index.css

Создать папки: components, hooks (если понадобятся), utils

Шаг 2. Стартовые данные и структура задачи
В App.jsx определить начальный массив задач (4 штуки с разными приоритетами, часть выполнена)

Каждая задача: { id: number, title: string, description: string, priority: 'high'|'medium'|'low', done: boolean }

Шаг 3. Отображение списка (базовое)
Создать компонент TaskList – принимает tasks и рендерит через map

Создать компонент TaskItem – отображает одну задачу: чекбокс, заголовок, приоритет, кнопки «Редактировать», «Удалить»

Передать в TaskItem колбэки onToggleDone, onDelete, onEdit из App

Шаг 4. Логика изменения состояния задач (подъём)
В App реализовать:

handleToggleDone(id)

handleDeleteTask(id)

handleAddTask(newTask)

handleEditTask(updatedTask)

Проверить, что UI обновляется

Шаг 5. Форма добавления задачи (управляемая)
Компонент AddTaskForm (локальный state: title, description, priority)

При отправке вызывать handleAddTask из пропсов, очищать форму

Использовать useRef для автофокуса на поле title при монтировании компонента

Шаг 6. Редактирование задачи
Создать компонент EditTaskModal (можно просто условный рендеринг внутри App или отдельный компонент)

При клике на «Редактировать» в TaskItem – открывается форма с текущими значениями задачи

После сохранения – закрыть модалку и обновить задачу через handleEditTask

Шаг 7. Фильтрация и поиск
В App добавить состояния: filterStatus ('all', 'active', 'completed'), searchQuery

Создать компонент FilterButtons – принимает текущий фильтр и колбэк для его изменения

Создать компонент SearchBar – управляемое поле поиска

Реализовать функцию getFilteredTasks() внутри App, которая сначала фильтрует по done (в зависимости от filterStatus), затем по вхождению searchQuery в title

Передавать в TaskList уже отфильтрованный массив

Шаг 8. Счётчик и кнопка «Очистить выполненные»
Отобразить количество невыполненных задач (tasks.filter(t => !t.done).length)

Кнопка «Очистить выполненные» вызывает handleClearCompleted (фильтрует tasks, оставляя только done === false)

Шаг 9. Сохранение в localStorage
useEffect при монтировании: прочитать localStorage.getItem('tasks'), распарсить, если есть – установить в tasks, иначе оставить начальные

useEffect с зависимостью [tasks]: сохранять tasks в localStorage (исключая первый рендер? Можно без проверки, просто перезаписывать)

Шаг 10. Использование useLayoutEffect
Добавить useLayoutEffect в компонент TaskList или App, который после добавления новой задачи (зависимость от tasks.length) измеряет высоту контейнера списка и выводит в консоль. Это покажет понимание синхронного выполнения до отрисовки.

Шаг 11. children – компонент-обёртка
Создать компонент Section (или Card), который принимает title (пропс) и children. Внутри рендерит <div className="section"> с заголовком и детьми.

Обернуть им, например, форму добавления и список задач.

Шаг 12. Стилизация (опционально)
Простой CSS, чтобы было понятно, что задачи выполненные зачёркнуты, приоритеты разного цвета.

Шаг 13. Проверка и отладка
Убедиться, что все хуки работают корректно, нет лишних рендеров, ключи уникальны.
